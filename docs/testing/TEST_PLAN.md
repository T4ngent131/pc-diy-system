# PC DIY 智能装机系统测试计划

## 1. 测试目标

验证 PC DIY 智能装机系统的核心业务链路稳定可用，重点覆盖注册登录、配件查询、兼容性校验、订单提交、库存出入库、知识库查询等功能。

## 2. 测试范围

### 功能测试

- 用户注册、登录、退出登录
- 配件分类查询、低库存查询、品牌/关键词筛选
- 自定义装机兼容性校验
- 推荐配置提交订单
- 订单创建、查询、状态更新
- 库存入库、出库、库存不足拦截
- 知识库文章列表、详情、分类查询

### 异常与边界测试

- 用户名长度不足
- 密码长度不足
- 重复注册
- 登录密码错误
- 空配置提交订单
- 配件 ID 不存在
- 出库数量小于等于 0
- 出库数量大于库存
- 查询结果为空

### 非测试范围

- 支付流程
- 第三方物流
- 真实短信验证码
- 高并发压测

## 3. 测试环境

- 前端：Vue 3 + Vite
- 后端：Spring Boot 3.2
- 数据库：MySQL，默认端口 `3307`
- 自动化：Node.js 内置测试框架、Fetch API

## 4. 执行命令

```bash
npm run test
npm run test:api
npm run test:all
```

接口测试默认访问：

```bash
http://localhost:8080
```

如需指定后端地址：

```bash
API_BASE_URL=http://localhost:8080 npm run test:api
```

## 5. 准入条件

- MySQL 已启动，端口为 `3307`
- `pc_diy` 数据库已建表并导入基础数据
- 后端服务已启动在 `8080`
- 前端依赖已安装

## 6. 退出标准

- 核心接口冒烟测试通过
- 兼容性规则单元测试通过
- 无阻塞级缺陷
- 关键业务链路可手动复现
